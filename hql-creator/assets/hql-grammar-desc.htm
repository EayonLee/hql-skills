子查询

Ø使用子查询的结果作为主查询的搜索过滤条件；

Painless

高性能、安全、可选类型、丰富语法的painless主要应用在where、eval命令上，即可下推到下层存储也可以本地运行

表达式、操作符、字面量等语法详细说明可以阅读Painless LanguageSpecification

部分修改

  1. 删除& ^ |运算符
  2. 布尔运算：and等同&& or等同|| not等同!
  3. 新增布尔操作符
    • xxx belong 信息组，内部转成_belong(信息组base64, xxx)
    • xxx like "aaa"，内部转成str_like(xxx, "aaa")
    • xxx rlike /正则/，内部转成xxx ==~ /正则/
    • exist xxx，内部转成xxx != null
  4. 中文标识符，根据属性字段转成对应英文；日志如有中文字段，可以使用doc['中文']
  5. IP字面量，内部转成字符串

where

  • 描述

    where命令使⽤完成Painless表达式筛选搜索结果。这些Painless表达式必须是布尔值表达式，返回true 或 false。where命令仅返回Painless表达式返回true的结果。

    Painless表达式可以下推ES执行，但长线存储暂不支持。

  • 语法

    where

    必要参数

    **

    • 语法：Painless LanguageSpecification

    • 描述：代表⽬标字段值的值、变量、运算符以及函数的组合，doc字段代表当前记录。

  • ⽤法

    where 命令使⽤的表达式语法与 eval命令相同。两个命令都将带引号的字符串解释为⽂字。如果字符串不带引号，则将其视为字段名称，您可以使⽤where命令来⽐较两个不同的字段。

  • ⽰例

    1. 使⽤ where 命令匹配 IP 地址或⼦⽹

    返回与 IP 匹配或在指定⼦⽹中的事件。

    ... | where 源地址 == "10.9.165.1" or ip_in(目的地址, "10.9.165.0/25")

    1. 使⽤ where 命令指定计算

    返回事件执行时间超过60秒的事件。

    ... | where 结束时间 - 开始时间 &gt; 60*1000

    1. 异常捕获
    ... | where try {
  return ipToLong(源地址) &gt; ipToLong('192.168.0.1');
} catch (Exception e) {
  return false;
}

eval

  • 描述

    eval命令painless表达式并将结果值放⼊⽬标字段中，若该⽬标字段与现有字段名称匹配，则会被覆盖，目标返回当前doc；或者返回新的Map对象。

  • 语法

    eval

    必要参数

    **

    • 语法：Painless Language Specification

    • 描述：代表⽬标字段值的值、变量、运算符以及函数的组合，doc字段代表当前记录。

  • ⽤法

    • 常规： eval命令要求指定⼀个字段名，⽤于接受要计算的表达式的结果。

    • 特殊： 表达式中使用到字段可能不存在，需要进行非null判断；或者try {} catch(Exception e) {}进行防守

  • ⽰例

    1. 创建包含计算结果的新字段

    ... | eval def _time = 接收时间 - 发生时间; def low = 事件名称.toLowerCase();long _ip = ipToLong(源地址)

    1. 创建新的返回对象
    ... | eval
def map = [:];
map['时间字段'] = 发生时间;
if (exist 源地址) {
  map.ip = 源地址
}
return map;
    1. 异常捕获
    ... | eval try {
  def _time = 接收时间 - 发生时间;
  def low = 事件名称.toLowerCase();
  long _ip = ipToLong(源地址);
} catch (Exception e) {
  return doc;
}

bucket

  • 描述

    通过对 field 值桶运算产生新的字段放到该事件中。

  • 语法

    bucket [...] [AS ]

    必要参数

    field

    • 语法： ``
    • 描述：指定⼀个字段名称。

    可选参数

    bucket-options

    • 语法： span | minspan | bins | start-end
    • 描述：分桶选项

    span

    • 语法： span = '' | ''
    • 描述：基于时间间隔分桶或者数字间隔分桶
      • 为格式的字符串，ms-毫秒 s-秒 m-分 h-时 d-天
        • 分桶算法：Math.floor(value / interval) * interval
      • 为整形数字字符串
        • 分桶算法：Math.floor((value - 0) / interval) * interval + 0

    minspan | bins

    • 语法： minspan = '' bins =
    • 描述：基于最小时间间隔和最大分桶数，再根据时间选择器起始结束时间（或者start-end时间戳参数），自动产生最佳时间间隔
    • 默认值：bins为15

    start-end

    • 语法： start= | end=
    • 描述：将数字型根据大于等于start和小于end的事件产生桶值为start的新字段

    newfield

    • 语法： ID
    • 描述：桶字段名
    • 默认值：__bucket

  • ⽤法

    为每笔事件产生一个分桶字段

  • ⽰例

    ⽰例1：返回每5分钟跨度内事件总数。

    ... | bucket span='5m' occur_time AS _time | stats count(id) BY _time

    ⽰例2：最近12小时(时间选择器)，产生最小间隔1分钟切最多50个桶，每个桶里事件总数

    ... | bucket minspan='1m' bins=50 occur_time | stats count(id) BY _occur_time_bucket

    ⽰例3：将源端口根据>=0且<100和>=100且<200产生桶值，其余不符合的事件不会产生桶值

    ... | bucket start=0 end=100 源端口 AS _range | bucket start=100 end=200 源端口 AS _range | ...

stats

  • 描述

    计算结果集的聚合统计，如平均数、计数和总和。类似于 SQL聚合。如果stats命令在没有BY⼦句的情况下使⽤，将只返回⼀⾏，也就是整个进来的结果集的聚合。如果使⽤了BY字段，将为BY字段中指定的每个唯⼀值返回⼀⾏。

  • 语法

    stats (stats-function(field) [AS field])... [BY field-list]

    必要参数

    stats-function

    • 语法： ()
    • 描述：统计聚合函数。

    可选参数

    by-clause

    • 语法： BY
    • 描述：根据一个或多个字段进行分组。

  • ⽤法

    统计函数中，如统计所有事件，字段为id

  • ⽰例

    1.统计事件总数

    ... | stats count(id)

    2.统计每个事件名称里的事件总数和源端口最大值

    ... | stats count(id) AS _count, max(源端口) BY 数据源,事件名称

sort

  • 描述

    sort 命令按指定字段对所有结果进⾏排序。

  • 语法

    sort ...

    必要参数

    **

    • 语法： ( - | + )? , ( - | + )? ...
    • 描述：排序所依据的字段的列表及其排序顺序。⽤减号 (-)表⽰降序，加号 ( + ) 表⽰升序。
    • 默认值：升序

  • ⽤法

    排序算法:

    1. 两值等同，或都为null时，如有下个字段排序，进行下个字段排序
    2. 非null值>null值
    3. 两值都为数字时，按数字比较
    4. 两值都为字符串时，按字典比较
    5. 一值为数字，另一直为字符串，字符串>数字
    6. 其余类型保持顺序

  • ⽰例

    先按 "_time" 字段以升序顺序对结果进⾏排序，再按 "host"值以降序顺序排序。

    ... | sort _time, -host

join

  • 描述

    使⽤ join 命令合并⼦搜索和主搜索的结果。多个join可以嵌套或串联在一起，最终生成一个宽表结果集。

  • 语法

    join [join-options]... left= right= where .=. [AND .=.]...

    必要参数

    subsearch

    • 语法： "[" subsearch "]"
    • 描述：用于构建Build表的子查询，优先于左侧查询启动；注意内存限制

    left

    • 语法： left=
    • 描述：为左侧查询起用的别名，以避免命名冲突。

    right

    • 语法： right=
    • 描述：为右子查询起用的别名，以避免命名冲突。

    where

    • 语法： .=. [ AND .=.]...
    • 描述：要join的左侧查询结果和右侧查询中结果的字段名称。必须指定别名和字段名称。例如：L.host=R.user。

    可选参数

    join-options

    • 语法： type=('inner' | 'left') | overwrite= | max=
    • 描述：join 命令的选项。

    type

    • 语法： type='inner' | 'left'
    • 描述：执⾏的连接类型。inner连接与 left连接的差别在于：。inner连接的结果不包含来⾃在⼦搜索中没有匹配项的事件；left连接的结果包括左侧查询中的所有事件。
    • 默认值：inner

    overwrite

    • 语法： overwrite=
    • 描述：来⾃右测⼦结果的字段是否覆盖来⾃左侧结果的字段（如果两者字段名相同）。
    • 默认值：true

    max

    • 语法： max=
    • 描述：指定每个左侧主搜索结果可以连接的最⼤右侧⼦搜索结果数。若设置为max=0，则⽆限制。
    • 默认值：0

  • ⽤法

    要返回⼀对多、多对⼀或多对多关系的匹配。max=1，意味着⼦搜索只返回⼦搜索中的⾸个搜索。将值设置为更⾼的数或设置为0（即不受限制），将返回⼦搜索的多值结果。

  • ⽰例

    index == 'event' | join max=1 left=l right=r where l.目的地址=r.ip_address [index == 'asset*' ]

map

  • 描述

    map 命令是⼀个针对每个输⼊事件或结果重新生成新的hql语句并执行的循环运算符。

  • 语法

    map [maxsearches=int]

    必要参数

    searchoption

    • 语法： ""

    • 描述：提供新产生的hql语句的模板。

    • 默认值：⽆

    可选参数

    maxsearches

    • 语法： maxsearches=

    • 描述：同时要运⾏的最⼤搜索数。

    • 默认值：1

  • ⽤法

    字段名称的变量

    map 命令⽀持替代输⼊结果中匹配字段名称的 $variable$ 字符串。例如，含 $count$ 字符串的搜索将⽤输⼊搜索结果中 count 字段的值替换变量。

  • ⽰例

    ... | map "index == 'alarm' | where 开始时间 &gt; $time$"

autoregress

  • 描述

    通过将 field 的⼀个或多个先前值复制到每个事件中，让这些事件为⾃动回归或移动平均值的计算做好准备。刚开始的⼀些事件由于之前的值不存在，会缺少之前值的增强功能。

  • 语法

    autoregress ([AS ] | -)

    必要参数

    field

    • 语法： ``

    • 描述：字段的名称。

    int

    • 语法： | -

    • 描述：指定从哪些之前的事件中复制值。

      • 可指定两种形式：

      • 单个整数和数字范围。若为单个整数，如3，autoregress 命令会将第三个先前事件的字段值复制到新的字段。

      • 若为数字范围，autoregress 命令会从先前事件的范围中复制字段值。例如，若您指定了⼀个数字范围，如 p=2-4，则第⼆个、第三个和第四个先前事件的字段值将被复制到新的字段中。

    可选参数

    newfield

    • 语法： ``

    • 描述：若将 int 设置为单个整数，newfield 参数将指定⼀个可把单个字段值复制到其中的字段名称。

  • 用法

    若未指定 newfield 参数，单个或多个值将被复制到名为 _p 的字段中。例如，若 p=2-4 且 field=count，字段名称将为 count_p2、count_p3 和 count_p4。

  • ⽰例

    例如用autoregress和tail算子，来支撑同比环比的计算

    index == "event" and 发生时间 &gt;= now(d-1d) and 发生时间 &lt; now(d) | stats count(id) AS count | append [index == "event" and 发生时间 &gt;= now(d) and 发生时间 &lt; now(d+1d) | stats count(id) AS count] | autoregress count 1 | eval double _count = count_p1 == null ? 0 : count_p1; count_prec = _count/count | tail 1 | fields count_prec
