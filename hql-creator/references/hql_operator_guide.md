# HQL 操作符说明

这份文档只解释操作符语义和渲染规则。  
它不是公共请求契约。

## 字段谓词操作

| 操作符族        | 标准 HQL 形态                     | 什么时候用                |
|-------------|-------------------------------|----------------------|
| 普通比较        | `字段 == "值"` / `字段 like "值"`   | 默认主路径，适用于普通标量条件      |
| `any_match` | `any_match(字段, ["值1", "值2"])` | 字段值需要匹配多个候选值中的任意一个   |
| `belong`    | `字段 belong 字段`                | 左侧字段需要判断是否属于某个业务归属字段 |

推荐选择顺序：

1. 普通比较
2. `like`
3. `any_match`
4. `belong`
5. `rlike`

也就是说，`rlike` 是最后选择的兜底操作符，不是默认主路径。

## 普通比较

支持：

- `==`
- `!=`
- `>`
- `>=`
- `<`
- `<=`
- `like`
- `rlike`

其中：

- `like` 用于通配匹配
- `rlike` 用于正则匹配
- `rlike` 的标准 HQL 右侧是 `/正则/`
- 请求里的 `value` 可写 `/正则/`，也可写纯正则文本；系统会统一渲染成 `/正则/`
- 如果普通比较、`like`、`any_match` 或 `belong` 已经能表达意图，就不要优先用 `rlike`

## `any_match`

标准渲染：

```hql
any_match(标签, ["高危", "外连"])
```

规则：

- 第二个参数永远渲染成列表
- 请求里的标准写法是数组
- 左侧字段可以是标量，也可以是数组
- 它的统一语义是“左侧字段值命中右侧候选值列表中的任意一个”

## `belong`

标准渲染：

```hql
源地址 belong 内网IP
```

典型场景：

- “最近七天有多少内网IP被模型研判为攻击成功？”
- “统计属于某信息组的请求数量”
- “查询属于某信息组的源地址”

规则：

- 标准渲染保持 plain `belong`
- 右侧必须是业务归属字段
- 右侧是字段，不是字面量
- 右侧是单个字段，不是数组
- 普通等值判断使用 comparison
- 当右侧意图是业务范围、业务名单、信息组、内网/外网范围，并且当前 source 存在对应的 `belong_rhs` 字段时，优先使用 `belong`

## 聚合指标函数

公共 metric 路径支持：

- `count`
- `avg`
- `dc`
- `estdc`
- `max`
- `min`
- `range`
- `sum`

默认聚合指标：

```hql
count(ID) AS 数量
```

硬规则：

- 标准输出不会生成 `count()` 或 `count(*)`
- 需要字段的函数必须提供字段

## 渲染规则

- 分组统计时才渲染 `BY <字段>`
- 总数统计绝不渲染 `BY`
- 分组排行会追加 `| sort -数量 | head N`
- `detail-topk` 会把分组统计放进子查询，最终只返回明细
